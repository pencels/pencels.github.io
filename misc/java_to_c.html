First, a comparison:

Let's use a familiar program to help get a grasp of what's going on in C...

“Hello World” in Java

/**
 * Hello.java
 * ============
 * Look at this wonderful Java code (:
 */
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello World!");
    }
}

“Hello World” in C

/*
 * Hello.c
 * ============
 * C doesn't look that bad either!
 */
int main(void) {
    printf("Hello World!\n");
    return 0;
}

The Java implementation should look familiar, so I’ll focus on the differences in the C code:

  • There is no class definition.
  • The main method has an int return type.
  • C looks much less verbose (Sorry, Java).

C actually has no class keyword, which is a result of it being an action-oriented language. This means that C focuses less on creating objects and more on defining functions to carry out actions. C programs are written mostly like the above—each method existing outside in the source code on its own. Despite being action-oriented, it still has the capability to create “objects,” but we’ll get into that later.

C also includes the definition of .h (header) files, which are similar to Java’s interfaces: they provide a public interface for use of the actual .c source code. The benefit of using header files is that they promote data hiding and abstraction of code.

The int return type is meant to indicate the exit code of the function. Usually, this is 0, indicating success (and why we return 0 at the end of main). The C’s main function also doesn't include static in its definition, since static has a different meaning in C from what it does in Java (more on that later).

Finally, we can see that C looks simpler than Java in a few ways: we use printf instead of System.out.println and int main instead of public static void main. These simplifications come with a cost in the long run, since C more intensely requires the programmer to manage memory allocation than Java does.

Some more differences:

Pointers

Both C and Java use these, but C makes the programmer use them more explicitly. This means if you have experience with Java, you already have experience with pointers; you just didn’t know you were using them until now.

Pointers really aren't that difficult to understand—they’re just variables that hold the address of other variables in memory. A common analogy to pointers are phone numbers: you keep someone’s phone number (i.e. their memory address) so that you can call them later (this is called ‘dereferencing’).

Here's an example of pointer declaration and initialization next to a regular old integer declaration: 

/* boring int declaration. */
int x = 42;

/* int pointer declaration! */
int * ptr;

/* int pointer initialized to point to x! */
int * xptr = &x;

It's important to note the * which indicates that the variable is really a pointer to an int, not an int type itself (though on some systems the memory address held is a 32-bit integer value).

You may also notice the & symbol on the third line. This is called the address-of operator, and it does exactly what its name says it does: it gets the address of the variable after it.

Now the third line makes sense. The int pointer xptr is being assigned the address of x from line 1. If we continue our phone-number analogy, this is the same as saving someone’s phone number to your address book. Now that we have x’s phone number, how do we call them? Here's some more code:

*xptr = 0; /* x is now 0 */

Now the * symbol has a different use here. Here we call it the dereference operator. It follows the memory address saved in xptr to where x lives, then assigns the memory space there the value 0. In our phone analogy, this is the same as calling the person whose phone number you have.
